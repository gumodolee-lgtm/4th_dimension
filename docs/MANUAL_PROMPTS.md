# 프롬프트 매뉴얼 (Prompts Manual)

Claude API에 전달하는 프롬프트의 작성, 관리, 버전 추적 규칙입니다.

## 프롬프트 위치

- 프롬프트 템플릿 파일: `scripts/prompts/` 디렉토리에 저장한다.
- 각 작업 유형별로 별도의 프롬프트 파일을 관리한다:
  - `classify-prompt.js` (또는 `.txt`) — 설교 분류용
  - `curriculum-prompt.js` — 교재 생성용
  - `lesson-prompt.js` — 개별 교안 생성용

## 규칙

### 분류 프롬프트 (Classification Prompt)
- 프롬프트는 4요소(생각/믿음/꿈/말)의 **명확한 정의**를 항상 포함해야 한다.
- 각 요소에 대한 판별 기준과 예시를 프롬프트 내에 제공한다.
- AI 응답 형식을 명시적으로 지정한다 (JSON 형식 강제):
  ```
  응답은 반드시 아래 JSON 형식을 따르시오:
  {
    "primary": "생각" | "믿음" | "꿈" | "말",
    "secondary": "생각" | "믿음" | "꿈" | "말" | null,
    "confidence": 0.0 ~ 1.0,
    "reasoning": "분류 근거 설명"
  }
  ```
- 동일한 설교에 대해 **일관된 분류 결과**가 나오도록 temperature를 낮게 설정한다 (0.0~0.3 권장).
- 분류 기준이 변경되면 기존 분류 결과와의 차이를 반드시 검토한다.

### 교재 생성 프롬프트 (Curriculum Prompt)
- 분류된 설교 데이터(`classified/`)를 컨텍스트로 반드시 포함한다.
- 교재 구조(주차별, 주제별)를 프롬프트에 명시한다.
- 성경 구절 참조가 정확한지 검증할 수 있도록 형식을 지정한다.
- 교재 분량과 난이도 기준을 프롬프트에 포함한다.

### 교안 생성 프롬프트 (Lesson Prompt)
- 해당 주차의 교재 데이터와 관련 설교를 모두 컨텍스트에 포함한다.
- 교안의 구성 요소(도입, 본문, 적용, 기도)를 명시한다.
- 한국어 표현의 자연스러움을 프롬프트에 요구한다.

### 버전 관리
- 프롬프트를 변경할 때마다 **변경 이유와 날짜**를 주석 또는 별도 로그에 기록한다.
- 주요 프롬프트 변경 시 이전 버전을 백업한다 (예: `classify-prompt.v1.js`).
- 프롬프트 변경 후 **샘플 5~10개**에 대해 결과를 비교 검증한다.
- 변경 이력 형식:
  ```javascript
  // Prompt Version History
  // v1.0 (2025-01-15) - 초기 분류 프롬프트
  // v1.1 (2025-02-01) - '말' 요소 판별 기준 강화
  // v2.0 (2025-03-10) - 보조 요소(secondary) 분류 추가
  ```

### 프롬프트 구조 가이드
- 역할 설정 (System prompt): AI의 역할과 전문성을 정의한다.
- 작업 지시 (Task): 수행할 작업을 명확히 기술한다.
- 입력 데이터 (Input): 설교 텍스트 또는 분류 결과를 전달한다.
- 출력 형식 (Output format): JSON 등 구조화된 형식을 강제한다.
- 제약 조건 (Constraints): 금지 사항이나 품질 기준을 명시한다.

## 안티패턴 (하지 말 것)

- 모호하거나 추상적인 지시를 프롬프트에 쓰지 않는다 (예: "잘 분류해 주세요" 대신 구체적 기준 제시).
- 설교 텍스트 없이 분류를 요청하지 않는다 (컨텍스트 누락 방지).
- 프롬프트에 4요소의 정의를 생략하지 않는다.
- temperature를 높게 설정하여 분류 일관성을 해치지 않는다.
- 프롬프트 변경을 기록 없이 적용하지 않는다.
- 하나의 프롬프트에 분류와 교재 생성을 동시에 요청하지 않는다 (작업별 분리).
